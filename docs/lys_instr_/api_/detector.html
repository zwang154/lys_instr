class lys_instr.MultiDetector.DetectorInterface(exposure=1, **kwargs)[source]

Bases: HardwareInterface

Abstract interface for detector devices.

This class provides background polling, threaded (worker) acquisition, and Qt signal management for detector state and data updates. Subclasses must implement device-specific methods: _get(), _stop(), and _isAlive(). _get() and _stop() should raise RuntimeError if the device is not responding or a communication error occurs. _isAlive() should always return the current alive state and should not raise RuntimeError that causes interruption. The updated signal should be emitted by the acquisition thread to inform when new data is acquired.

Parameters:

  • exposure (float or None) – The initial exposure time. If the device does not support the exposure time, None should be set.

  • **kwargs – Additional keyword arguments passed to QThread.

Abstract interface for multi-detector devices.

This class extends DetectorInterface to support detectors that acquire multi-dimensional data frames. Subclasses should implement device-specific logic for data acquisition and shape reporting.
